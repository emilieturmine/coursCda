But de la réalisation:

Créer la Base De Données à l’aide du langage SQL de description de données (Data Description Language

Techniques abordées :

Démarrer l’analyseur de requêtes SQL  Utiliser l’aide en ligne de MariaDB  A l’aide de l’éditeur, écrire un script SQL pour : o Créer une base de données : CREATE DATABASE o Créer des tables : CREATE TABLE o Créer des clés primaires et étrangères :  Interpréter des éventuels messages d’erreur

Travail à réaliser:

1) A partir du MPD ci-dessus définissez le dictionnaire des données (système de codification, type de données) pour identifier le type de chaque attribut.

Code Libellé Type Contraintes Règle de calcul
per_num numéro de la personne int auto-increment
per_nom nom de la personne varchar(255)
per_prenom prenom de la personne varchar(255)
per_adresse adresse de la personne varchar(255)
per_ville ville de la personne varchar(255)
gro_num numero du groupe int auto increment
gro_libelle libelle du groupe varchar(255)

2) Repérez les clés primaires et les clés étrangères sur les différentes tables.

Pour la table "Personne" on retrouve "per_num" en clé primaire , pas de clé étrangere

Pour la table "Groupe" , on retrouve "gro_num" en clé primaire

Pour la table Appartient créer lors du passage du modele entités association au mdp, on a "per_num" et "gro_num" toutes cles etrangeres issu des tables "Personne" et "Groupe" qui ENSEMBLE forme la clés primaire de la table.

3) Codez en SQL les trois tables, votre script doit s'exécuter sans erreur.

CREATE DATABASE exo1CBDD; USE exo1CBDD; CREATE TABLE Personne( per_num INT AUTO_INCREMENT NOT NULL, per_nom varchar(255), per_prenom varchar(255), per_adresse varchar(255), per_ville varchar(255), PRIMARY KEY (per_num) ) ; CREATE TABLE Groupe( gro_num INT AUTO_INCREMENT NOT NULL, gro_libelle varchar(255), PRIMARY KEY (gro_num) ); CREATE TABLE Appartient ( per_num int, gro_num int, PRIMARY KEY(per_num,gro_num), FOREIGN KEY (per_num) REFERENCES Personne(per_num), FOREIGN KEY (gro_num) REFERENCES Groupe(gro_num) );
