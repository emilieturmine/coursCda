Pour l'association nommée "passe" entre les entités "Client" et "Commande", puisque les cardinalités maximale sont 1 et n ,la transformation de l'association en MCD se fait donc par la suppression de l'association et l'ajout d'une clé étrangere "n_client" dans la table "Commande" avec une flèche reliant la table "Client" pour indiquer l'origine de celle -çi.

Pour l'association nommée "SeComposeDe" entre les entités "Commande" et "Article" , puisque les cardinalités maximale sont n et n ,la transformation de l'association en MCD se fait donc par la création d'une nouvelle table remplaçant l'association avec l'ajout de deux cles étrangeres "N_Commande" et "N_Article" issus respectivement des tables "Commande" et "Article" ( origine representé par une fleche).Ensemble ces deux clés étrangeres deviennent la clé primaire de cette nouveele table.

Pour la table "SeComposeDe",on retrouve "N_Commande" et "N_Article" en clé secondaire,elles composent ensemble la clé primaire de la table

CREATE DATABASE evaluation; USE evaluation; CREATE TABLE Client( n_Client INT AUTO_INCREMENT NOT NULL, NomClient VARCHAR(255), PrenomClient VARCHAR(255), PRIMARY KEY(n_Client) ); CREATE TABLE Commande( N_Commande INT AUTO_INCREMENT NOT NULL, DateCommande DATE, MontantCommande CURRENCY, n_Client INT NOT NULL, PRIMARY KEY(N_Commande), FOREIGN KEY(n_Client) REFERENCES Client(n_Client) ); CREATE TABLE Article( N_Article INT AUTO_INCREMENT NOT NULL, DesignationArticle VARCHAR(255), PUArticle CURRENCY, PRIMARY KEY(N_Article) ); CREATE TABLE Facture( N_Commande INT, N_Article INT, Qte INT, TauxTva DECIMAL(15,2), PRIMARY KEY(N_Commande, N_Article), FOREIGN KEY(N_Commande) REFERENCES Commande(N_Commande), FOREIGN KEY(N_Article) REFERENCES Article(N_Article) );
